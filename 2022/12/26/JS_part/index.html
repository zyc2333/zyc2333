var\let\const的区别

-var 
    是ES3定义变量的方式；
    在作用域内声明前置
    可重复声明
    没有块级作用域
- let
    let和const是ES6定义变量的方式
    先声明再适用（无声明前置）
    不可重复声明（大括号里可重复声明）
    有块级作用域
- const
    同let
    声明时需要立即初始化
    **初始化后不能再修改！**

函数声明和函数表达式的区别

函数表达式必须前置才不会报错，而函数声明不用前置也不会报错。

词法作用域、作用域链

1.作用域是值和表达式能被访问的执行上下文
2.如果值和表达式不在当前作用域，就无法访问
3.作用域有层级，子作用域可访问祖先作用域变量
4.作用域种类：
    全局作用域；
    函数作用域；
    块级作用域（{}创建）；
    模块作用域（模块模式下）。
5.词法作用域
    被创建函数所在的作用域。
6.作用域链
    先从自己作用域找，再去被创建的作用域找，直到最上层。
7.声明前置
8.TDZ
    CONST和let声明的变量在当前作用域的开头（'{'）到声明的区域是暂时性死区，不可以用该变量。

Map对象

1.是键对值的集合。Map中的一个键只能出现一次。任何值都可作为键和值。

2.Object和Map的区别：
Map键可以是任何类型，Object只能是字符串和Symbol；
Map的键是有序的，Object是无序的；
Map可以获取size，Object需要自己计算；
Map性能更好；
Map是可迭代的可用for...of.Object是不可迭代的，可用for...in遍历；

专用字符

具有特殊意义的字符

( [ { \ ^ $ | } ] ) * + .

单字符匹配

[abcd] //匹配一个字符，是abcd中的任意一个
[0-9] //匹配一个0-9的数字
[a-zA-Z]//匹配一个不限制大小写的字母
[^abc]//不是a/b/c的字符

预定义匹配

. //等价于[^\r\n]，匹配一个除回车和换行以外的任意字符
\d //等价于[0-9]，匹配一个数字字符
\D //等价于[^0-9],匹配一个非数字字符
\s //等价于[空格\t\tn\r\v\f],匹配一个空白字符
\S //匹配一个非空白字符
\w //等价于[a-zA-Z_0-9],匹配一个字母、数字、下划线
\W //匹配一个非单词字符（例如：he-llo=>he,llo）

0个或多个

? //前面的字符出现最多1次
+     // 大于等于1次
*     // 大于等于0
{n}   //出现n次
{n,m}//[n,m]次
{n,} //大于等于n次

边界

/^xyz/ 以xyz开头(边界！不是单字符匹配，区别在于[])
/abc$/ 以abc结尾
/\babc\b/ 匹配是单词的abc（左右两侧是字符串开头、结尾、中横线、空格）
/\Babc\B/ 匹配不是单词的abc

^开头，$结尾。

贪婪模式VS非贪婪模式

默认是贪婪模式。尽可能多去匹配。如果是非贪婪模式，需要在量词(+ * .)后面加？号。

分组

用与把匹配的部分拿出来

注意区分.match和.matchAll方法：
1.match返回值是一个数组，如果没有任何匹配项则返回null；matchAll返回迭代器，要想查看其结果需要遍历迭代器。
2.match匹配到第一个匹配项后即停止匹配；matchAll会匹配出字符串中所有的匹配项。
