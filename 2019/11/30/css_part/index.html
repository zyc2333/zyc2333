布局模式

  • **普通流(Normal flow)**：所有元素都是普通流中的一部分，包括块(block)布局和行(inline)布局。

  • 表格布局：仅用来设置表格。

  • 浮动布局：用来让元素左右排放。过时。

  • 定位布局：用来定位元素，将某元素固定在某位置。

  • 多列布局：用来设置多列。不常用。

  • 弹性盒：常用。

  • 网格布局：通过二维网格设置布局。可用。

标准盒模型

  • Content：内容。

  • Padding: 内容到边框的距离。

  • Border：边框。

画圆：radius = 宽高的一半

  • Margin：和其他元素的距离。

块级盒子 vs. 行级盒子

块级(block-level)盒子

  • 会在水平方向上扩展并占据父容器在该方向上的所有可用空间；下一个盒子会换到新行
  • width 、height、padding、margin、border都生效
  • 块级标签

行级(inline-level)盒子

  • 多个行级盒子在一行顺序摆放，放不下会自动换行

  • 对于仅展示文本的标签，如 a、span、em、strong、label，
    width 和 height 属性设置无效。padding、margin左右有效，上下margin无效（上下padding会撑开自己的边框和背景但不占空间）

  • 行级标签

inline block

既有块级特性：能设置width、height、margin、padding、border；又有行级特性：仅占据自身空间、多个在一行排列。

small quiz——inline元素缝隙问题

  • TAB、 LF(/n 换行，老打字机打印头下移一行)、 CR(/r 回车，老打字机打印头到纸张开头)、 SPACE 都是空白字符

  • 多个连续的空白字符会合并成一个空格，而空格也占据一个字符的空间

  • 去除方法：方法1. HTML里inline元素间去除空格和换行；方法2: 用flex/grid布局；方法3: 容器font-size设置0；

外边距折叠

  • 块级元素的上外边距和下外边距有时会合并为一个外边距，其大小取其中的最大者

  • 浮动元素和绝对定位元素的外边距不会折叠

  • 相邻折叠：相邻的两个块级元素之间的外边距会合并，取其中最大值。

  • 嵌套折叠：如果在父元素与其嵌套的后代元素间没有border、padding、行内内容，也没有创建块格式化上下文，父子外边距会折叠。

    容器加overflow: hidden 或者 auto 、border、padding可阻止折叠。

  • 空块折叠：如果一个块级元素中不包含任何内容，并且在其 margin-top 与 margin-bottom 之间没有边框、内边距、行内内容、height、min-height 将两者分开，则该元素的上下外边距会折叠。

  • 左右外边距不折叠

定位

  • position: static、relative、absolute、fixed、sticky

  • top、left、right、bottom

  • z-index

  • static

    1.指定元素在普通流中使用默认布局位置，等同于不加

    2.一般用于覆盖设置过其它定位方式的场景

    3.此时 top, right, bottom, left 和 z-index 属性无效

    4.日常所说的定位元素不包括 position: static

  • relative

    1.未脱离普通流（为元素预留空间），元素本身在
    普通流中的位置并未发生变化，只是视觉上变化经常。

    2.作为absolute定位位置参考。

    3.微调。

  • absolute（定位元素）

    1.指定元素脱离普通流（不为元素预留空间），相对于最近的非 static 定位的祖先元素来进行偏移

    2.场景：下拉菜单、弹窗、输入框内图标，以及其它任何可能产生“覆盖”的场景

  • fixed（定位元素）

    1.从普通流中完全脱离，相对于浏览器窗口进行定位

    2.一定要设置top 或者 bottom，否则可能出现无法展示的情况

    3.为啥加width: 100%？为啥加背景色？

    绝对定位（absolute和fixed会收缩到内容的宽度）。不加背景色是透明的（为什么不让他透明？）

  • sticky

    1.一般情况下（未碰触视窗顶部）处于普通流中等同于position: static

    2.当其碰到视窗顶部会被拦住，等同于成fixed效果

    3.一定要设置top 或者 bottom

    4.如果出现多个sticky，放置在同一容器内，会出现下一个sticky元素把上一个“覆盖”的效果

    5.如果出现多个sticky，放置在并列的多个块级容器内，会出现下一个sticky元素把上一个“推走”的效果

  • NOTICE

    1.定位元素仅指代设置 absolute 和 fixed 的元素

    2.absolute元素必须设置参考点(给某个祖先元素设置relative)

    3.定位元素宽度会“收缩”，一般需要设置固定宽度或者宽度100%

    4.设置为定位元素的行内元素拥有块级特性，可设置宽高

    5.定位元素脱离普通流，可相互覆盖，通过z-index设置层级

    6.position包含哪5个，absolute和fixed定位是相对于什么来偏移。
